> 队伍：忍不了小队 | 配合《01-系统模块架构设计.md》一起看。
> 这份文档回答两个问题：**产品做成什么技术形态**，以及 **9 个模块，3 天里哪些真做、哪些只做 UI 展示，怎么分步骤落地**。

## 一、技术形态：Web App（移动端优先响应式），不做小程序

**结论**：做一个响应式网页应用，手机/电脑都能打开，重点适配移动端。不做微信小程序。

**原因**：
- 小程序要注册 AppID、配置服务器域名白名单，调用 AI API 还需提前在后台报备域名，3 天时间大概率来不及
- 网页部署是分钟级的（Vercel/Netlify 一键上线），拿到一个链接就能发给评委，评委用手机当场就能试用，不用装任何东西
- 移动端优先的响应式网页在手机上打开体验和小程序几乎没有区别，视觉上不吃亏，但开发和调试速度快得多

## 二、版本规划总览

| 版本 | 时间预期 | 核心目标 | 一句话状态 |
|---|---|---|---|
| MVP（最小可实现版） | 黑客松 3 天内 | 核心闭环真实可用 + 全部功能都有 UI 可点 | "小而完整，故事讲得清楚" |
| 进阶版 | 赛后 1-2 个月 | 把 MVP 里的"假数据"模块变成真实功能 | "从单人工具变成有内容生态的产品" |
| 完整版 | 长期愿景 | 完整社区+沉浸式+语音全场景 | "自我生长的应对能力社区" |

**核心原则**：MVP 不是"砍掉大部分功能"，而是"**每个模块都在同一个 App 里可以点开体验，只是背后数据的真假程度不同**"——评委看到的是一个完整的产品，而不是"能用的部分 + PPT 里的补充说明"两张皮。

## 三、MVP 版本详细设计（黑客松 3 天）

### 目标
对齐报名表里的预期产出：**一个可在线体验的 Web 应用 MVP，包含 10-15 个匿名化沟通案例、场景输入、AI 策略拆解、三种语气的回应建议及一轮模拟练习**。这条链路必须又稳又亮眼，是 demo 的绝对核心。

> 口径说明：这里的"匿名化沟通案例"，团队内部约定实现为"基于真实常见沟通模式原创提炼的复合案例"（见下方风险修订），这比"抓取真实帖子后去除姓名"更彻底地做到了匿名——因为压根不存在可追溯的原始来源，对外表述保持一致即可，不影响报名表内容的准确性。

### 模块状态总览表

| 模块 | MVP 状态 | 说明 |
|---|---|---|
| ⑦ 话术生成助手 | ✅ 真实功能 | 完整开发，调用真实 API，输入→分类→生成三版本回应 |
| ⑨ 情景练习（职场场景） | ✅ 真实功能 | 完整开发一个场景的选项分支剧情（基础版） |
| ② 真实案例库 / 案例广场浏览 | ✅ 真实功能 | 10-15 条匿名化种子案例，这本身就是预期产出的一部分 |
| ① 应对框架库 | ✅ 真实功能（后台） | 手工整理 15-20 条，支撑生成引擎，用户不直接看到 |
| ③ 法律支持库 | 🔶 静态展示（安全设计） | 法律参考做成人工预审的静态卡片，AI 不现场生成法律内容，仅做标签匹配后插入固定文本；MVP 只做 2-3 条团队有把握的条目，明确标注"仅供参考，非法律意见" |
| ⑧ 社区广场（发布/评论/点赞互动） | 🔶 UI 展示（假数据） | 界面能点，点击后提示"社区功能即将开放" |
| ⑨ 情景练习（家庭/社交/购物场景） | 🔶 UI 展示（假数据） | 场景卡片能点开，展示设计好的开场情境，但走不完整分支逻辑，标注"体验版" |
| ⑥ 语音交互引擎 + 沉浸式互动 | 🔶 UI 展示（假数据） | 界面和交互按钮做出来，点击后提示"敬请期待" |

**关键技巧**：🔶 的部分用**真实前端组件渲染硬编码的 JSON 假数据**，不是静态截图/图片——照样能点、有动效、有交互反馈，成本比截图高不了多少，但体验上"活"很多，评委也能感受到你把整个产品想透了。

**诚实原则**：demo 时要主动说清楚哪些是真实跑通的、哪些是概念展示，不要让人误以为全都是真的。这个产品本身的主题就是"边界感和坦诚表达"，讲清楚状态反而是一种呼应，被现场问出来含糊其辞会很减分。

**⚠️ 三项风险修订**（详细设计见《01-系统模块架构设计.md》对应模块）：
1. **法律支持库**：不让 AI 现场生成法律内容，只做静态人工审核内容的标签匹配插入，避免"AI 编法条"被现场问穿；同时只对真正触及法律边界的情境（职场权益、名誉权、消费者权益、治安管理等）展示法律参考，纯道德/家庭/情感类情境不强行关联法律内容
2. **案例库**：种子案例改为团队基于真实常见模式原创撰写的"复合案例"，不对应具体真实帖子/博主，避免版权和隐私边界问题
3. **场景分类器 + 生成引擎**：新增"关系与权力维度"判断，每个语气版本附一句适用提示（比如对上级/长辈时优先推荐更稳妥的版本），避免无差别推荐"犀利反击版"

### 3 天开发节奏建议

**Day 1：打地基**
- 整理框架库（15-20 条）和案例库（10-15 条基于真实常见模式原创撰写的复合案例，不对应具体真实帖子/博主）——这是纯人力活，越早开始越好，建议团队里熟悉真实案例的人优先做这个
- 跑通场景分类器 + 内容生成引擎的 prompt，用几条真实情境反复调试输出质量
- 确定整体页面结构和路由（话术助手 / 案例广场 / 情景练习 / 法律库入口 / 社区广场），先把"骨架"和导航都搭出来，方便后面并行开发

**Day 2：搭产品**
- 完成话术生成助手的完整交互（输入→分类→生成→展示三版本）
- 完成情景练习职场场景的完整分支逻辑
- 案例广场浏览页接入真实案例库数据
- 其余模块（社区互动、多场景练习、语音、沉浸式）先用假数据把 UI 界面和交互动效做出来，能点、能看，暂不接真实逻辑

**Day 3：打磨和演练**
- 准备 2-3 条人工审核好的法律参考静态卡片，接入标签匹配逻辑（不经过生成模型）
- 整体走查体验，修复明显 bug，统一视觉风格
- 准备 demo 稿：核心闭环怎么演示、UI 展示部分怎么讲清楚"这是规划、当前是概念版"

## 四、MVP 核心用户流程（分步骤）

1. 用户打开应用，看到"说不出话的时候，让我帮你说"的入口
2. 输入情境，例如"同事在会上当众说我方案想当然"
3. （后台）系统判断：职场 / 当众否定 / 高紧急度 / 关系为平级同事
4. （后台）匹配框架库"命名行为—设立边界—转移主动权"，检索出 2-3 条相关复合案例
5. 生成 3 个语气版本回应，每个附"为什么有效"和一句"适用提示"（结合关系维度判断），如命中静态法律库条目会附一条预审的法律参考卡片
6. 用户复制/收藏喜欢的版本
7. 用户点进"练习"，选择职场场景，进入一段选项分支剧情，尝试不同应对看不同结局；点开其他场景卡片会看到"体验版"标注
8. 用户浏览案例广场，看看 10-15 条基于真实沟通模式提炼的复合案例，加深"这个东西贴近真实场景"的信任感；尝试点赞/评论会看到"社区功能即将开放"的提示

## 五、进阶版详细设计（赛后 1-2 个月）

**核心思路**：把 MVP 里标 🔶 的模块，逐一变成 ✅ 真实功能。

| 模块 | 增强内容 |
|---|---|
| 真实案例库 | 开放用户上传，AI 自动拆解结构化；建立简单审核队列 |
| 社区广场 | 从展示态升级为真实可互动：发布情境/案例、评论、点赞；高赞内容自动进入待收录队列 |
| 法律支持库 | 覆盖领域扩展到 4 类（职场/名誉权/消费者权益/治安管理），结构化关联到案例库 |
| 情景练习 | 场景类型全部做实：职场、家庭（对父母）、社交餐饮、购物消费，均为完整选项分支 |
| 内容生成引擎 | 引入用户历史使用记录，让生成结果逐渐贴近用户自己的说话风格 |

**用户流程更新**：在 MVP 流程基础上，增加"广场共建"闭环——用户在广场看到别人的求助情境 → 给出自己的应对建议（评论）→ 好的建议被点赞达到阈值 → 系统标记"建议收录进案例库"→ 官方简单审核后正式入库，原作者获得"贡献案例"的成就感反馈。

## 六、完整版详细设计（长期愿景）

### 完整功能全景
- **情景练习**：新增沉浸式自由对话版本，AI 实时扮演对方、动态调整剧情走向；全场景覆盖 + 语音交互（ASR/TTS）默认开启，练习结束后有风格分析报告
- **社区广场**：发展为完整社交生态——话题标签、关注机制、"高频冲突"周榜/月榜，甚至可以有专业人士（心理咨询师/律师）入驻答疑
- **法律支持库**：与真实法律从业者合作校对内容，标注权威来源，避免"AI 编法条"的风险
- **个性化能力**：AI 逐渐了解每个用户的性格和语言习惯，生成的回应越来越像"用户自己会说的话"，而不是通用模板

### 长期可探索方向（非本次重点，仅作为愿景提及）
- 与心理咨询/法律咨询服务的轻量级导流合作
- 沉淀出的高质量应对框架和案例，反哺成"职场沟通/情绪边界"类知识付费内容

## 七、三类典型用户旅程（文字流程图）

### 旅程 A：求助型（最高频，MVP 重点打磨）
遇到情境 → 打开话术助手 → 输入情境 → 获得 3 版本回应+解析 → 使用/收藏 → （进阶版）事后分享到广场

### 旅程 B：练习型
选择练习入口 → 选场景类型 → 选难度（基础版/进阶版）→ 完成一轮练习 → 获得反馈 → 重复练习巩固信心

### 旅程 C：共建型（进阶版起）
逛广场 → 看到别人的情境 → 评论/共创方案 → 自己上传案例 → 案例被点赞认可 → 正式收录进案例库，获得成就感

## 八、Demo 展示建议

现场时间有限，建议按这个节奏讲：

1. **30 秒痛点引入**：一个真实感很强的"被为难到哑口无言"的场景
2. **2 分钟核心演示**：现场输入一个情境，展示分类→生成→3 版本回应+解析的完整链路（这是最能体现技术含量和产品思路的部分）
3. **1 分钟练习演示**：跑一遍职场场景的选项分支练习，体现"不只是给答案，还能练习"
4. **1 分钟完整功能巡览**：点开社区广场、其他场景练习、语音/沉浸式入口，明确说明"这些是产品规划的完整形态，UI 已经做出来，当前版本背后是演示数据，核心生成逻辑已经打通"
5. **收尾**：回到"生存时间"赛道命题——这个产品想帮女性把"被动挨说、事后崩溃"的时间，重新变成"当下就能体面回应"的主动权
