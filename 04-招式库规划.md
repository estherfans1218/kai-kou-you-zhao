# 开口有招 · 招式库规划 v1

## 一、招式库不是“怼人语录库”

招式库解决的是：面对不同局型，用户先判断自己要解决什么，再选择一种可复用的表达结构。具体句子可以变化，底层结构必须稳定。

完整决策链为：

> 安全判断 → 识别局型 → 明确目标 → 判断权力关系 → 选择招式 → 准备下一回合 → 必要时留痕、升级或离场

任何招式都不以“吵赢”为唯一结果。成功可以是拿到明确答复、守住边界、让责任可确认、保留证据，或者安全结束对话。

## 二、理论骨架

招式库由四组方法共同支撑，不把任何一种理论当成所有问题的万能答案。

1. **基础逻辑与论证识别**：区分事实与评价、前提与结论、举证责任、错误二选一、偷换概念和过度概括。用于识别对方的话哪里没有落到事实。
2. **DBT 人际效能框架**：困难对话前先判断本局优先保护的是“目标、关系还是自尊”；DEAR MAN 提供描述事实、表达影响、明确主张、说明收益、保持聚焦和协商的结构。主要用于提出请求和拒绝。
3. **非暴力沟通 NVC**：观察、感受、需要、请求四个组件，尤其强调把评价改写为可观察事实，把要求变为具体可执行的请求。用于减少不必要升级，不等于一味温柔退让。
4. **原则性谈判**：区分人和问题、关注利益而不是立场、寻找选项、使用客观标准，并提前明确谈不成时的替代方案（BATNA）。用于职场协调、消费争议和需要解决具体问题的场景。

四个辅助框架只用于增强招式，不单独堆出新的“理论招式”：

5. **自信表达训练**：区分被动、攻击和坚定表达，帮助招式做到清楚但不羞辱。
6. **心理抗拒理论**：检查命令式、限制选择的语言是否会无谓激起抵触；“给出下一步”必须提供真实选择，而不是伪装成选择的威胁。
7. **会话分析与修复机制**：把招式设计成连续回合，处理澄清、误解修复、话题转移、自我修正和结束对话。
8. **实施意图与压力练习**：把招式转化为“如果对方做 X，我就做 Y”的预案，通过分支剧情和自由对练形成临场反应。

哲学中的言语行为理论用于内部标注一句话是在陈述、请求、拒绝、承诺还是结束对话；博弈论只辅助长期互动、可信后果和退出选项，不直接生成日常台词。

## 三、选招前的四个问题

每个案例进入招式匹配前，先回答四个问题：

| 判断维度 | 可选答案 | 对输出的影响 |
|---|---|---|
| 当前安全吗 | 安全 / 有升级风险 / 需要立即离场 | 不安全时不生成逞强反击，优先退场、求助、留证 |
| 本局目标是什么 | 拿结果 / 保关系 / 守边界 / 澄清事实 / 结束对话 | 决定招式和语气，而不是默认“怼回去” |
| 权力关系如何 | 对等 / 对方权力更高 / 用户权力更高 / 陌生人 | 对上级、长辈和陌生人给不同风险提示 |
| 对方用了什么模式 | 模糊指责 / 扣帽子 / 道德施压 / 模糊加码 / 转移议题 / 拒绝解决 / 持续纠缠 | 匹配对应招式，并预测下一回合 |

## 四、MVP 八个核心招式

### 01 把话落地

- **识别信号**：“你一直都……”“大家都觉得……”“你态度有问题。”
- **目标**：把模糊评价转换成可以核对的事实。
- **公式**：请具体到“哪一次、哪件事、哪个结果”。
- **依据**：事实与评价的区分、主张者承担举证责任。
- **不适用**：对方正在人身威胁或明显失控时，不留在现场辩论事实。

### 02 拒绝接帽

- **识别信号**：“你不答应就是自私。”“你这么做就是不合群。”
- **目标**：承认具体事件，但拒绝从一次行为推导整个人格。
- **公式**：我愿意讨论这件具体的事，但不接受把它概括成对我的人格评价。
- **依据**：过度概括、错误二选一、结论缺少前提。
- **不适用**：如果用户确实造成具体损害，需要先承担具体责任，再限制结论范围。

### 03 把代价摆桌上

- **识别信号**：临时加活、所有任务都说重要、要求“顺手做一下”。
- **目标**：让新增任务对应的时间、资源和延期责任可见。
- **公式**：现在有 A、B；加入 C 后，您希望暂停或顺延哪一个？
- **依据**：资源约束、明确取舍、DEAR MAN 的明确主张与协商。
- **不适用**：真正紧急且已有明确应急职责的场景，先处置再复盘资源安排。

### 04 拉回原题

- **识别信号**：把退款问题说成服务态度问题，把时间问题说成忠诚问题，把证据问题说成信任问题。
- **目标**：识别偷换概念，持续把对话拉回待解决事项。
- **公式**：我们可以稍后讨论 X，现在先确认 Y 的处理方式和时间。
- **依据**：偷换概念、区分人和问题、保持聚焦。
- **不适用**：对方补充的是确实会改变结论的新事实时，应先处理新事实。

### 05 边界三句

- **识别信号**：冒犯性玩笑、反复打听隐私、越界评论、未经同意替用户做决定。
- **目标**：把不舒服变成明确、可执行的边界。
- **公式**：当你做 X（事实）→ 这会造成 Y（影响）→ 请停止/改为 Z（具体请求）。
- **依据**：NVC 的观察与具体请求；DEAR MAN 的描述、表达和主张。
- **不适用**：持续骚扰者不一定值得反复解释；说过一次后可直接结束互动。

### 06 给出下一步

- **识别信号**：双方不断解释立场，但没有人提出怎么解决。
- **目标**：把争论变成选择和行动。
- **公式**：现在可以 A，也可以 B；如果都不能，请告诉我负责人和明确时间。
- **依据**：关注利益而非立场、创造可选方案、请求必须具体可执行。
- **不适用**：不能把本应由对方承担的法定或工作责任包装成用户让步。

### 07 用标准说话

- **识别信号**：“我们一直都是这样。”“我觉得没问题。”“别人都能接受。”
- **目标**：从意气和权力较量转向双方可核对的标准。
- **公式**：我们按订单、书面约定、时间记录、公开规则或行业标准确认。
- **依据**：原则性谈判中的客观标准。
- **不适用**：法律、劳动和医疗等高风险内容只能引用人工审核的静态资料，AI 不现场编法条。

### 08 留痕、升级或离场

- **识别信号**：多轮沟通无进展、对方否认已说内容、持续辱骂、威胁或拖延。
- **目标**：停止无效回合，把行动转向记录、第三方或替代方案。
- **公式**：我把刚才确认的事项发成文字；如果在 X 时间前无法处理，我会联系 Y/采用 Z。
- **依据**：BATNA（谈不成时的最佳替代方案）、对话安全边界。
- **不适用**：升级路径必须真实可执行，不用虚假的“我认识谁”或空洞威胁。

## 五、每张招式卡必须包含的字段

```ts
type MoveCard = {
  id: string;
  name: string;                 // 用户记得住的招式名
  pattern: string;              // 对方正在使用的沟通模式
  goal: string;                 // 这一招要解决什么
  formula: string;              // 可复用结构，不是固定台词
  whyItWorks: string;           // 作用机制
  theory: string[];             // 理论依据
  suitablePower: string[];      // 适合的权力关系
  riskBoundary: string;         // 什么时候不要用
  firstReply: string;           // 第一回合示例
  branches: Array<{
    label: string;              // 配合、继续模糊、转移攻击、拒绝处理等
    opponent: string;
    response: string;
    note: string;
    closing: {
      opponent: string;
      response: string;
      note: string;
    };
  }>;
  practicePrompt: string;       // 费曼复述或情景练习题
};
```

用户界面只显示一句简明的“为什么有效”；完整理论依据放在可展开的“方法依据”中，避免一屏字太多。

每张招式卡不是一句金句，而是一棵最小决策树：第一回合给出标准动作；第二回合让用户从四种典型反应中选择；第三回合完成解决、修复、留痕或退出。同一招必须展示“对方配合时如何收束”和“对方继续施压时如何切换”，避免把强硬重复误写成万能策略。

## 六、内容生产顺序

MVP 不按“职场、家庭、消费”各写一堆孤立金句，而是先完成 8 个招式，再让每一招跨场景验证：

1. 每个招式先写 1 张标准教学卡，共 8 张。
2. 每个招式再写 2 个不同关系的案例，至少一个连续三回合，共 16 个案例。
3. 场景总量中优先保证：职场 6、消费 4、家庭 3、社交/群聊 3。
4. 每个案例必须标注目标、权力关系、风险和不适用条件。
5. 最后用真实局投稿验证：同一招能否迁移到新的场景，而不是只适合一个固定句子。

## 七、质量审核清单

- 这条内容是在解决问题，还是单纯让人觉得“怼得爽”？
- 回应中有没有加入素材里不存在的事实？
- 是否给了用户一个明确下一步？
- 对方继续施压时，第二回合还能成立吗？
- 对上级、亲密关系和陌生人是否错误地使用了同一风险等级？
- 是否把“温和”误写成讨好，把“坚定”误写成羞辱？
- 是否在不安全场景里鼓励用户留下争辩？
- 理论名词能否真正解释这句话为什么有效，而不是事后贴标签？
